## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-to-input spacing |
| sm | 8px | Compact element spacing, input internal padding |
| md | 16px | Default gap between form fields, card internal padding |
| lg | 24px | Section padding within Card components |
| xl | 32px | Gap between form section Cards |
| 2xl | 48px | Page top/bottom padding |
| 3xl | 64px | Not used in this phase |

Exceptions: Image thumbnails in the Images Card use 12px gap (3 * 4px) between grid items for tighter visual grouping.

## Form Layout Contract

### Vehicle Form Sections (single scrollable page)

1. **Vehicle Information** (Card) -- Make, Model, Year, VIN, Mileage, Exterior Color, Interior Color, Body Style, Transmission, Fuel Type, Drivetrain, Condition, Status
2. **Pricing & Costs** (Card) -- List Price, Purchase Price, Purchase Date
3. **Description & Features** (Card) -- Description (Textarea), Features (Textarea or comma-separated)
4. **Images** (Card) -- Upload button, thumbnail grid, reorder arrows, delete buttons, primary star

### Field Grid

- Use 2-column grid (`grid grid-cols-1 sm:grid-cols-2 gap-4`) for related short fields (Make + Model, Year + Mileage, Ext Color + Int Color)
- Full-width for Textarea fields (Description, Features, Notes)
- Full-width for VIN (long value)

### Customer Form

Single Card: First Name, Last Name, Email, Phone, Address, City, State, Zip

### Lead Form

Single Card: Customer selector or name fields, Vehicle selector (optional), Source, Status, Notes

### Deal Form Sections

1. **Deal Info** (Card) -- Deal type/status, SearchableSelect for Customer, SearchableSelect for Vehicle
2. **Pricing** (Card) -- Sale Price, Trade-in Value, Down Payment, Interest Rate, Loan Term. Live-calculated read-only fields: Monthly Payment, Total Cost, Gross Profit

## Interaction Contract

### Field-Level Validation

| Trigger | Behavior |
|---------|----------|
| First submit | Validate all fields. Show errors on invalid fields. Scroll to first error. |
| After first submit, on blur | Re-validate the blurred field. Remove error if fixed, show error if still invalid. |
| On change (after first submit) | Do NOT re-validate on every keystroke. Only on blur. |
| Valid field after fix | Remove `border-destructive`, hide error message immediately on blur. |

### Error Display

- Border: `border-destructive` on the Input/Select/Textarea
- Message: `<p className="text-xs text-destructive mt-1">{message}</p>` directly below the field
- No error summary banner at top of form
- Scroll to first error: `element.scrollIntoView({ behavior: 'smooth', block: 'center' })`

### Submit Button States

| State | Label | Appearance |
|-------|-------|------------|
| Idle | "Save Vehicle" / "Save Customer" / "Save Lead" / "Save Deal" | `Button` variant="default" (primary navy) |
| Submitting | "Saving..." | `Button` disabled, with `Loader2` icon spinning (lucide) |
| Create mode | "Add Vehicle" / "Add Customer" / "Add Lead" / "Add Deal" | Same as idle |
| Edit mode | "Save Changes" | Same as idle |

### Post-Submission

| Outcome | Behavior |
|---------|----------|
| Success (create) | `toast.promise()` success message, redirect to entity list page |
| Success (edit) | `toast.promise()` success message, redirect to entity list page |
| Server validation error | Map Zod field paths to form field errors, display inline |
| Network/server error | `toast.error()` with generic message, form stays populated |

### Unsaved Changes

- Track form dirty state via @tanstack/react-form
- `beforeunload` event listener when form is dirty
- No in-app modal -- browser native prompt only

### Image Management

| Action | Interaction |
|--------|-------------|
| Upload | File input button "Add Images" at top of Images Card. Accepts multiple files. |
| Reorder | Up/Down arrow icon buttons on each thumbnail. Disabled at boundary positions. |
| Set primary | Star icon (lucide `Star`) on each thumbnail. Filled star = primary, outline = not primary. Click to set. Only one primary at a time. |
| Delete | Trash icon button (lucide `Trash2`) on each thumbnail. Opens Dialog: "Delete this image? This cannot be undone." with Cancel + Delete buttons. |
| Thumbnail grid | CSS grid, 4 columns on desktop (`grid-cols-2 sm:grid-cols-3 md:grid-cols-4`), `gap-3` (12px). Each thumbnail is 1:1 aspect ratio with `object-cover`. |

### Searchable Select (Deal Form)

| State | Behavior |
|-------|----------|
| Closed | Shows selected value or placeholder text in trigger button |
| Open | Popover with Command input at top, scrollable list below |
| Searching | Filters list items as user types. "No results found." if empty. |
| Selection | Click item to select. Popover closes. Trigger shows selected name. |
| Display format | Customer: "First Last (email)". Vehicle: "Year Make Model (stock #)" |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Labels | Every input has a `<Label htmlFor={id}>` with matching `id` on the input |
| Error association | Error messages use `aria-describedby` on input, `id` on error paragraph |
| Invalid state | `aria-invalid="true"` on inputs with validation errors |
| Required fields | `aria-required="true"` on required inputs |
| Focus management | Scroll to first error field on submit. Focus the field after scroll. |
| Image alt text | Thumbnails use `alt="Vehicle image {position}"` |
| Dialog | shadcn Dialog handles focus trap and Escape-to-close |
| Button disabled state | `aria-disabled` and `disabled` attribute during submission |
| Keyboard navigation | All form controls reachable via Tab. Enter submits form. |

## Responsive Contract

| Breakpoint | Layout Change |
|------------|---------------|
| < 640px (mobile) | Single column form fields. Image grid 2 columns. Full-width Cards. |
| >= 640px (sm) | 2-column grid for paired fields. Image grid 3 columns. |
| >= 768px (md) | Image grid 4 columns. |
| >= 1024px (lg) | No additional changes -- forms are max-width constrained within admin layout. |

Form max-width: `max-w-4xl` (896px) centered within the admin content area.
