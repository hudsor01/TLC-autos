## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, tight padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section inner padding, card body spacing |
| xl | 32px | Layout gaps between major elements |
| 2xl | 48px | Section vertical padding (minimum via clamp) |
| 3xl | 64px | Page-level section breaks |

Exceptions: Section vertical padding uses `clamp(3rem, 6vw, 6rem)` (48px-96px) via the existing `.section-padding` utility class. Touch targets for mobile CTA buttons use 44px minimum height (later phases).

### 60/30/10 Distribution

| Role | Percentage | Value | Usage |
|------|-----------|-------|-------|
| Dominant | 60% | #faf9f7 (--background) | Page background, body surfaces |
| Secondary surface | 30% | #ffffff (--card), #f5f3f0 (--muted) | Cards, elevated surfaces, muted sections |
| Accent | 10% | #a81225 (--secondary burgundy) | CTAs, brand highlights, price badges, active links |

Accent (--secondary, burgundy #a81225) reserved for:
- Primary CTA buttons (bg-secondary)
- Price display text on vehicle cards
- Active/current navigation indicators
- Brand-highlighted text spans in headlines
- Hover states use opacity: `hover:bg-secondary/90`

### Semantic Colors (unchanged values)

| Token | Value | Contrast on #faf9f7 | Usage |

If success/warning are used as body text, darken to: success #15803d, warning #b45309.

### Design Rules

- No gradients anywhere -- flat, clean surfaces only
- No dark hero or CTA sections -- ALL LIGHT
- No `dark:` Tailwind prefix usage anywhere in codebase

## Animation Contract

### Entrance Animations (scroll-reveal)

| Class | Duration | Easing | Transform | Usage |
|-------|----------|--------|-----------|-------|
| `.animate-fade-up` | 400ms | ease-out | opacity 0->1, translateY(12px->0) | Default scroll-reveal for sections |
| `.animate-fade-in` | 400ms | ease-out | opacity 0->1 | Subtle entrance, no translation |
| `.animate-slide-up` | 500ms | ease-out | opacity 0->1, translateY(20px->0) | Hero elements, larger entrance |

All animations use `animation-fill-mode: both` so elements hold final state.

### Stagger Delays

| Class | Value | Usage |
|-------|-------|-------|
| `.delay-75` | 75ms | 1st item in group |
| `.delay-150` | 150ms | 2nd item |
| `.delay-225` | 225ms | 3rd item |
| `.delay-300` | 300ms | 4th item |

### Hover Transitions

| Class | Duration | Easing | Effect | Usage |
|-------|----------|--------|--------|-------|
| `.hover-lift` | 150ms | ease-out | translateY(-2px) + --shadow-card-hover | Cards, interactive surfaces |

### Performance Rules

- Only animate `transform` and `opacity` (GPU-composited properties)
- Existing `prefers-reduced-motion` block (globals.css Section 6) kills all animation/transition durations automatically
- No JavaScript animation in this phase -- CSS-only utilities
- tw-animate-css provides shadcn component animations (Dialog, Popover, Command enter/exit)

### tw-animate-css Integration

Import order in globals.css (must be after tailwindcss):
```
@import "tailwindcss";
@import "tw-animate-css";
```

## Homepage --hero Migration (Scope Boundary)

When `--hero` and `--hero-foreground` are removed from `:root`, the homepage `page.tsx` must be updated in the same task to prevent breakage:

| Current | Replacement |
|---------|-------------|
| `style={{ backgroundColor: "var(--hero)" }}` | `className="bg-muted"` |
| `text-white` (in hero/CTA sections) | `text-foreground` |
| Hardcoded `#c8102e` (6 occurrences) | `text-secondary` / `bg-secondary` |
| Hardcoded `#ef4444` (3 occurrences) | `text-secondary` |
| Hardcoded `#a80d26` (2 occurrences) | `hover:bg-secondary/90` |
| Gradient accent lines | Remove entirely (no gradients rule) |

The layout redesign is Phase 7. This phase only makes the homepage render correctly with the new light-only palette.
