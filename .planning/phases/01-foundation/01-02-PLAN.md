1. Create `scripts/` directory if it does not exist.

2. Create `scripts/seed.ts` as a standalone TypeScript script (runs with `bun run scripts/seed.ts` or `npx tsx scripts/seed.ts`). The script must:

   a. **Import and create admin client directly** (do NOT import from `@/lib/supabase/admin` -- path aliases don't work in standalone scripts):
      ```typescript
      import { createClient } from "@supabase/supabase-js";
      import type { Database } from "../src/types/database.types";

      const supabase = createClient<Database>(
        process.env.NEXT_PUBLIC_SUPABASE_URL!,
        process.env.SUPABASE_SECRET_KEY!,
        { auth: { autoRefreshToken: false, persistSession: false } }
      );
      ```

   b. **Clean existing data before seeding** (in correct FK order):
      - Delete from follow_ups, deals, leads, vehicle_costs, vehicle_images, customers, vehicles
      - Also clean Storage bucket: list all files in `vehicle-images` bucket, delete them all
      - Print "[Seed] Cleaned existing data" after cleanup

   c. **Fetch existing auth.users** to get real user IDs for assigned_to/created_by:
      ```typescript
      const { data: authResponse } = await supabase.auth.admin.listUsers();
      const staffUsers = authResponse?.users ?? [];
      ```
      If no users found, print a warning but continue (assigned_to/created_by will be null).

   d. **Seed ~20 vehicles** with realistic used car data:
      - Mix of makes: Toyota, Honda, Ford, Chevrolet, Nissan, BMW, Mercedes-Benz, Hyundai, Kia, Dodge
      - Years: 2015-2023 range
      - Statuses: mix of "available" (majority), "sold", "pending"
      - Realistic pricing: purchase_price $5,000-$35,000, selling_price 15-25% markup
      - Stock numbers: "TLC-001" through "TLC-020"
      - VINs: generate realistic-looking 17-char VINs (e.g., "1HGCM82633A" + 6 random digits)
      - Fill all fields: trim, body_style, exterior_color, interior_color, mileage (20k-150k), fuel_type, transmission, engine, cylinders, drivetrain, description, features (JSON array of strings)
      - Use `.insert(vehicleData).select()` to get back the inserted rows with their IDs

   e. **Upload vehicle images to Supabase Storage:**
      - For each vehicle, fetch 2-3 images from `https://picsum.photos/800/600` (returns random photos -- acceptable for development seed data)
      - Upload each fetched image to `vehicle-images` bucket at path `vehicles/{vehicle.id}/{index}.jpg`
      - Insert corresponding `vehicle_images` records with `url` set to the Supabase Storage public URL: `{SUPABASE_URL}/storage/v1/object/public/vehicle-images/vehicles/{vehicle.id}/{index}.jpg`
      - Set `is_primary: true` on the first image, `sort_order` matching index
      - Add a small delay between image fetches (100ms) to avoid rate limiting picsum.photos
      - Print progress: "[Seed] Uploaded images for vehicle {stock_number}"

   f. **Seed ~15 customers** with realistic names:
      - Mix of first/last names, emails ({first}.{last}@example.com), phone numbers (555-xxx-xxxx format)
      - Some with full address (city/state/zip), some partial
      - Use `.insert(customerData).select()` to get IDs

   g. **Seed ~25 leads** with varying statuses:
      - Statuses: "new", "contacted", "qualified", "converted", "lost"
      - Sources: "website", "phone", "walk-in", "referral"
      - Some linked to customers (customer_id), some standalone (just first/last/email/phone)
      - `assigned_to`: distribute across staff user IDs from step (c). Use round-robin: `staffUsers[i % staffUsers.length]?.id ?? null`
      - vehicle_interest: reference vehicle makes/models from the seeded vehicles
      - Use `.insert(leadData).select()` to get IDs

   h. **Seed ~10 deals** with realistic financial data:
      - Link to vehicles (use "sold" status vehicles) and customers
      - sale_type: "cash", "finance" (mix)
      - Status: "pending", "approved", "funded", "delivered"
      - Realistic financials: selling_price matching vehicle, tax_rate 6.25, calculated tax_amount, doc_fee $150, title_fee $33, registration_fee $75
      - For financed deals: apr 3.9-8.9%, term 36/48/60/72 months, calculated monthly_payment
      - `created_by`: distribute across staff user IDs same as leads
      - Use `.insert(dealData).select()` to get IDs

   i. **Seed follow-ups** for some leads (~15 follow-ups):
      - Types: "note", "call", "email", "appointment"
      - Mix of completed and pending
      - `created_by`: same staff user who owns the parent lead
      - Some with due_date in the past (completed), some in the future (pending)

   j. **Print summary** at the end:
      ```
      [Seed] Complete!
      [Seed]   Vehicles: 20
      [Seed]   Vehicle Images: ~50
      [Seed]   Customers: 15
      [Seed]   Leads: 25
      [Seed]   Deals: 10
      [Seed]   Follow-ups: 15
      ```

   ```
   No local images are stored in this directory.

   ```

4. Ensure the script handles errors gracefully:
   - If SUPABASE_SECRET_KEY is not set, print error and exit with code 1
   - If NEXT_PUBLIC_SUPABASE_URL is not set, print error and exit with code 1
   - Wrap each major section in try/catch, print the error, and continue to next section
   - Image upload failures should log a warning but not halt the script
