## Copywriting Contract

Phase 1 is infrastructure-focused. The only user-facing copy changes are the toast notification messages and the logout interaction.

| Element | Copy |
|---------|------|
| Logout button label | "Sign Out" (existing, no change) |
| Toast: CRUD success | "{Entity} {action}ed" (e.g. "Vehicle created", "Lead updated") |
| Toast: CRUD error | "Failed to {action} {entity}" (e.g. "Failed to delete vehicle") |
| Toast: CRUD loading | "{Action}ing {entity}..." (e.g. "Creating vehicle...", "Deleting lead...") |
| Toast: generic error | "Something went wrong. Please try again." |
| Empty state | N/A -- Phase 1 does not introduce any list views or empty states |
| Destructive confirmation | N/A -- Logout has no confirmation dialog per CONTEXT.md decision |

Toast copy rules:
- Always use sentence case, not title case
- Always name the entity being acted on
- Loading state uses present participle ("Creating...", "Updating...", "Deleting...")
- Success state uses past tense ("Created", "Updated", "Deleted")
- Error state uses "Failed to {verb}" pattern

Source: CONTEXT.md locked decisions (toast.promise pattern, no logout confirmation)

## Toast Interaction Contract

Position: bottom-right (per CONTEXT.md)
Dismiss: auto-dismiss after default duration (5 seconds) + manual close button (`closeButton` prop)
Rich colors: enabled (`richColors` prop) -- maps to semantic color tokens
Animation: default Sonner slide-in from right
Stacking: default Sonner stacking (max 3 visible)

### Toaster Configuration

```tsx
<Toaster position="bottom-right" closeButton richColors />
```

### Usage Pattern (for all future CRUD operations)

```tsx
toast.promise(asyncOperation(), {
  loading: "Creating vehicle...",
  success: "Vehicle created",
  error: "Failed to create vehicle",
});
```

Theme integration: Sonner with `richColors` uses its own semantic colors. To match the design system, the Toaster's `toastOptions` can apply the project's CSS variables. Default Sonner styling is acceptable for Phase 1; theme refinement is at Claude's discretion per CONTEXT.md.

Source: CONTEXT.md locked decisions, RESEARCH.md Pattern 3

## Logout Interaction Contract

Trigger: "Sign Out" button in admin sidebar bottom section
Behavior: immediate signOut, no confirmation dialog
Redirect: `/` (public homepage, not `/admin/login`)
Post-redirect: `router.refresh()` to clear cached server data
