Rewrite `src/app/contact/contact-form.tsx` to add Zod validation while keeping the existing form structure:

1. Add imports: `import { z } from "zod"` and `import { toast } from "sonner"`

2. Define contactFormSchema at module level using Zod 4 syntax:
   ```typescript
   const contactFormSchema = z.object({
     firstName: z.string().min(1, { error: "First name is required" }),
     lastName: z.string().min(1, { error: "Last name is required" }),
     email: z.email({ error: "Enter a valid email address" }),
     phone: z.string().optional(),
     subject: z.string().min(1, { error: "Please select a subject" }),
     message: z.string().min(10, { error: "Message must be at least 10 characters" }),
   });
   ```

3. Define FieldError type and state:
   ```typescript
   type FormErrors = Partial<Record<keyof z.infer<typeof contactFormSchema>, string>>;
   ```
   - State: `const [errors, setErrors] = useState<FormErrors>({})`
   - State: `const [submitted, setSubmitted] = useState(false)`
   - State: `const [submitting, setSubmitting] = useState(false)`

4. Create `validateField` function for on-blur validation:
   ```typescript
   function validateField(name: keyof FormErrors, value: string) {
     const fieldSchema = contactFormSchema.shape[name];
     const result = fieldSchema.safeParse(value);
     setErrors((prev) => ({
       ...prev,
       [name]: result.success ? undefined : result.error.issues[0]?.message,
     }));
   }
   ```

5. Update onSubmit handler:
   - Build form values object from FormData (firstName, lastName, email, phone, subject, message)
   - Run `contactFormSchema.safeParse(formValues)`
   - If not success: extract field errors from result.error.flatten().fieldErrors, set first error for each field into errors state, return early (do NOT submit)
   - If success: use `toast.promise` wrapping the fetch call:
     ```typescript
     toast.promise(
       fetch("/api/leads", {
         method: "POST",
         headers: { "Content-Type": "application/json" },
         body: JSON.stringify({
           firstName: formValues.firstName,
           lastName: formValues.lastName,
           email: formValues.email,
           phone: formValues.phone || "",
           source: "website",
           status: "new",
           vehicleInterest: formValues.subject === "vehicle-inquiry" ? "Vehicle inquiry from website" : "",
           notes: `Subject: ${formValues.subject}\n\n${formValues.message}`,
         }),
       }).then((res) => {
         if (!res.ok) throw new Error("Failed to submit");
         setSubmitted(true);
       }),
       {
         loading: "Sending your message...",
         success: "Message sent successfully!",
         error: "Failed to send message. Please try again.",
       }
     );
     ```

6. Update form fields to show errors:
   - Each field group gets an onBlur handler: `onBlur={(e) => validateField("fieldName", e.target.value)}`
   - Below each Input/Select/Textarea, add error display:
     ```tsx
     {errors.fieldName && (
       <p className="mt-1 text-sm text-destructive">{errors.fieldName}</p>
     )}
     ```
   - Add `aria-invalid={!!errors.fieldName}` to each input
   - Add `className={cn("...", errors.fieldName && "border-destructive")}` for red border on error (import cn from @/lib/utils)

7. Keep the success state UI exactly as-is (checkmark SVG, "Message Sent!", "Send Another Message" button). When clicking "Send Another Message", also reset errors: `setErrors({})`.
